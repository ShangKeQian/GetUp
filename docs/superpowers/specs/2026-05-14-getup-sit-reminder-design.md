# GetUp - 久坐提醒软件设计文档

## 概述

GetUp 是一个 Windows 系统托盘应用，检测用户是否在电脑前，连续坐 30 分钟后弹出全屏遮罩提醒起身活动。目标是帮助用户避免久坐，保护健康。

## 核心功能

1. **检测用户在场状态**：支持两种检测方式（可切换）
   - 键盘/鼠标活动监听
   - 红外摄像头帧差法
2. **计时管理**：有人时累计计时，无人时重置
3. **提醒机制**：全屏半透明遮罩 + 倒计时
4. **系统托盘**：静默运行，提供控制入口

## 技术栈

- **语言**：Python 3.10+
- **键盘/鼠标监听**：pynput
- **摄像头**：opencv-python
- **GUI**：tkinter（内置）
- **打包**：PyInstaller → 单个 exe

## 状态机

```
空闲 → 检测到有人 → 计时中 → 30分钟到 → 显示遮罩
  ↑                                      │
  └─── 2分钟无人 ←──── 计时中 ←─────────┘
                        (遮罩关闭后)
```

- **空闲**：启动后 / 2分钟无活动后
- **计时中**：检测到有人，开始累计
- **遮罩**：30分钟到了，显示提醒；检测到有人回来时暂停倒计时

## 模块设计

### 1. 检测模块

#### 键盘/鼠标检测器

- 使用 `pynput.mouse.Listener` 和 `pynput.keyboard.Listener`
- 底层调用 Windows `SetWindowsHookEx` 系统钩子，事件驱动
- 资源占用：CPU ~0% 空闲，内存 ~1-2MB
- 每次检测到活动，更新"最后活动时间"戳
- 判断逻辑：`当前时间 - 最后活动时间 < idle_timeout` → "有人"

#### 红外摄像头检测器

- 使用 `cv2.VideoCapture` 打开摄像头
- 帧差法（不需要 AI 模型，资源占用极低）：
  - 每 3-5 秒采集一帧
  - 转灰度，与上一帧做像素级差异计算
  - 差异超过阈值 → 画面有变化 → 有人
  - 连续多帧无差异 → 可能无人
- 红外摄像头优势：不受光线影响，暗光/夜晚也能检测

### 2. 计时引擎

- 维护状态：`idle` / `timing` / `overlay`
- `idle` 状态：等待检测到有人，切换到 `timing`
- `timing` 状态：每秒递减计时，检测到无人（2分钟无活动）→ 重置回 `idle`
- `overlay` 状态：显示遮罩，检测到有人 → 暂停倒计时；遮罩关闭 → 回到 `timing`（从 0 开始）

### 3. 提醒界面（全屏遮罩）

- `tkinter.Toplevel` 全屏窗口
- `overrideredirect(True)` 去掉标题栏
- 半透明黑色背景（`attributes('-alpha', 0.85)`）
- 居中显示倒计时文字："请起身活动 X 分钟" + 大号倒计时
- 关闭方式：右上角关闭按钮 / 按 Esc 键
- 自动消失：倒计时结束后自动关闭
- 遮罩显示期间：
  - 检测到有人回来 → 暂停倒计时
  - 检测到无人（2分钟）→ 遮罩自动消失

### 4. 系统托盘

- 使用 `pystray` 库（或 tkinter 内置托盘支持）
- 菜单项：
  - 启动/停止计时
  - 检测模式切换（键盘鼠标 / 摄像头 / 同时启用）
  - 设置（作息时间间隔、活动时间、摄像头选择）
  - 退出

### 5. 配置管理

保存到 `config.json`，支持通过托盘菜单修改：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| work_minutes | 30 | 连续坐多久后提醒（分钟） |
| break_minutes | 2 | 建议活动时间（分钟） |
| idle_timeout | 2 | 多久无活动算离开（分钟） |
| detection_mode | "keyboard_mouse" | 检测模式：keyboard_mouse / camera / both |
| camera_index | 0 | 摄像头设备编号 |

## 打包

使用 PyInstaller 打包为单个 exe：

```bash
pyinstaller --onefile --windowed --name GetUp main.py
```

## 预期资源占用

- 内存：~30-50MB（Python 运行时 + 依赖库）
- CPU：空闲时 ~0%，检测事件时微秒级处理
- 磁盘：打包后 ~30-50MB
